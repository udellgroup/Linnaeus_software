What is Linnaeus?

Linnaeus is a tool for detecting equivalence between iterative optimization algorithms. Many algorithms that appear different on the surface — different names, different papers, different update rules — turn out to compute exactly the same iterates. Linnaeus finds these hidden equivalences automatically.

How It Works

Every iterative oracle-based algorithm can be expressed as a linear dynamical system in feedback with a nonlinearity (the oracle). Linnaeus parses your algorithm's update equations, computes a transfer function H(z), and compares it against a library of known methods:

  1. Parse: Your update equations are converted to a state-space realization (A, B, C, D).
  2. Transfer function: H(z) = D + C(zI − A)−1B is computed via z-transform.
  3. Equivalence test: H(z) is compared against all library entries using three tests: oracle equivalence (identical H(z)), shift equivalence (H(z) related by a diagonal shift), and LFT equivalence (related oracles, e.g., prox vs. conjugate prox).

Equation Syntax

Enter one update equation per line using [k], [k+1], [k-1] for time indices. Use * for explicit multiplication.

# Nesterov's Accelerated Method
y[k] = x[k] + beta * (x[k] - x[k-1])
x[k+1] = y[k] - alpha * grad_f(y[k])

# Douglas-Rachford Splitting
x1[k+1] = prox_f(x3[k])
x2[k+1] = prox_g(2 * x1[k+1] - x3[k])
x3[k+1] = x3[k] + x2[k+1] - x1[k+1]

About the Project

Linnaeus is based on the paper “Algebraic characterization of equivalence between optimization algorithms” by Laurent Lessard and Madeleine Udell (arXiv:2501.04972, 2025). The name honors Carl Linnaeus, the father of biological taxonomy, reflecting the tool's goal of classifying algorithms.
